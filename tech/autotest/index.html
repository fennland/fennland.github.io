    • tidevice+Python facebook-wda+weditor：Windows平台对iOS设备进行自动化测试的框架，用于测试BelaHome iOS App，前两个启动自动化，weditor定位元素同上
    • 雷电模拟器：自动化测试可以用的Android模拟器
    • scrcpy：自动化测试可用，在电脑上投屏并点击Android设备屏幕的工具
    • 向日葵：远程连接到测试间电脑
      • 建议本地自测用例3遍通过后再向日葵到测试间验证；

    • Postman：调试接口请求与响应 *

  3. 软测相关：

    • MobaXTerm或SecureCRT：看个人喜好二选一，有Macro捷径，PC的IP配置后可以有SSH/Telnet连接的预设，所有IP都有，预设不用再手动输IP和端口号；

    • 固件NAS

    • tftpd32：FTP工具，在Android或者Linux设备到电脑上，通过ftp互传文件

    • HFS：http文件服务器，一般用于挂载升级计划里的固件包

  4. 个人推荐安装的小工具：

    • LANDrop：iOS/Android/PC/macOS之间互传文件，速度取决于路由器局域网上限。手机拍完复现步骤的视频，不用企业微信压缩再上传到电脑了
    • Listary：Windows全局文件搜索，Everything其实也很好用，但是Listary更像macOS的Spotlight一样无感。当开发过来找你看导出的日志的时候，直接Ctrl两下搜log马上就出来
    • Typora：Markdown编辑器，可视化实时编辑，谁用谁知道
    • Windows Terminal：Microsoft商店搜索，比CMD和Powershell好用
    • IIS：用来在电脑开FTP，这样只用把文件丢到FTP文件夹，设备端执行ftpget的捷径就可以拉去电脑上传过去的linux so库了

*   **包名**：com.*******.panel; com.******.voiceassistant; com.******.upgradeui

*   **功能**：(1) 面板UI。包含家居设备控制（继电器、各类智能家居能力、监控、能耗等）、联系人对讲、家居云功能（安防告警、手动自动场景等）、机身功能（闹钟计时器、音乐、屏保壁纸、设置与升级等）——*可参照禅道机型版本测试单的“**关键功能点**”用例，在测试中快速了解面板功能。

    (3) 升级界面。设备接收到家居云下发版本更新（强制或用户确认更新）后，拉起的一个升级界面。

*   **常见测试用例**：关键功能点、本地模式


##### 2. 工厂测试    ******* FactoryTest

*   **包名：**com.******.factorytest
*   **功能：**整机测试、主板测试等。在工作中一般大版本会有厂测用例需要执行，主板测试需要拆下主板、上对应的主板工装、烧录固件后再执行主板测试；整机测试则在机器上正常测试。一般包含WIFI、蓝牙、Zigbee等10余项测试点，有些会自动测试，有些需要手动或Python自动化压测，全部通过为正常。

终端设备, 工程WEB¶

##### 1. 几种角色



*   **家庭中心**（AKA 室内机/智能家居面板）：家庭内的主控设备。
    *   家庭内任意设备发起设备探测（门口机等），都是通过家庭中心扫描的
    *   接收告警、物业通知等都经过家庭中心转发
    *   家庭中心不能从云上的家庭配置内单独删除，只能替换。家庭中心恢复出厂或掉线时，子网关设备为无家庭中心状态，相关能力被禁用。家庭项目删除时，家庭中心、子网关都会被移除并需要恢复出厂设置。
*   **子网关**（AKA 室内机/智能家居面板）：家庭内的被控设备。
*   **门口机**：智能对讲的对讲门禁机器，包含摄像头的对讲和开门系统，使用“继电器”（DTMF信令）控制开门
    *   带视频的对拨、开门
    *   视频监控
*   **管理机**：可粗略认为是一种特别的门口机设备
    *   接收告警和物业通知

    *   对拨


##### 2. 两个状态

*   **上云**：设备接入到家居云和对讲云的云服务，影响SIP通话、App等云业务
    *   **通话**：默认为SIP通话，即每个用户/家庭/设备分配一个10位数SIP账号，呼叫到对讲云SIP服务器后转发给呼叫对象的SIP
    *   **App**：依次创建——代理商账号->安装商->社区/单住户项目->家庭->家庭主账号，从邮件获得账号密码。
*   **本地模式**：设备在本地局域网内，安装配置时**手动选择“家庭中心”或“子网关”角色**（1个家庭对应1个家庭中心和N个子网关），**扫描配对组网**，从而在本地局域网内**建立1-N家庭关系**（1个局域网内可有N个家庭）。
    *   **通话**：通话走IP协议，默认不会在对讲云服务器上注册SIP通话账户。门口机可以直接输入IP呼叫到本地局域网内的任意设备，设备不需要在SIP注册并配置为联系人。
    *   **App**：使用本地模式体验账户扫码登录体验。需要保证本地局域网有在线的本地家庭中心

Linux与Android设备¶

常用操作¶

  1. 连接SSH：

    • 前置：：打开远程SSH，Android设备用SSH连接，Linux设备用Telnet

    • 捷径连接：配置网段并将电脑固定IP后，MobaXTerm文件夹下有网段的捷径，导入，选择设备对应的SSH/Telnet分类快速连接；

    • 手动连接：新建SSH/Telnet连接，注意部分设备端口不是22/23。

  2. 烧写MAC：

    • 前置：连接SSH，开通测试人员专属的MAC前缀
    • Android SSH终端：远程到设备终端，输入mkdir -p /info/*******; echo "eth0=C2:11:24:06:07:18\nserial=1122" > /info/*******/interface;chmod 777 -R /info/*******/其中，C2:11是测试人员标识前缀，后4个段建议写为当前年月日时。完成后reboot重启
    • Linux设备：设置****键值对为HWA_0:C2:11:24:06:07:18（可能有冒号）

  3. 日志与调试相关：

    • Android SSH终端流式打印日志：

      • 进入SSH终端，logcat -v threadtime流式打印日志。
      • 可后接| grep '...'来筛选相关内容；
      • logcat *******:E *:S 阻止其他标记日志，仅查询*标记的错误级别的日志

    • Linux Telnet终端打印最新的历史日志：tail -F /tmp/********

    • Busybox 录制操作与回放：

      • 设置环境变量：export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/system/lib:/app/lib
      • 启动录制：/app/bin/**busybox touchkey_ctrl start
      • 结束录制：/app/bin/**busybox touchkey_ctrl stop
      • 移动录制文件：mv /tmp/tp_record.txt /config/testcase_name.txt
      • 录制回放:/app/bin/**busybox touchkey_replay /config/testcase_name.txt
      • 循环回放：watch -n 30 /app/bin/**busybox touchkey_replay /config/testcase_name.txt；-n：后面跟的时间，几秒循环回放录制

      可以PC通过adb push D:\ftpshare\testcase_name.txt sdcard将用例录制传到设备上，再移动到config目录

    • 上传崩溃日志到FTP服务器：ftpget -u ******* -p ************* ************** /info/******_androidlog.sh android/script/******_androidlog.sh; chmod 777 /info/******_androidlog.sh ; /info/******_androidlog.sh

      • 如果有出现崩溃的问题 网页导出log的同时，后台命令执行一下这句，然后bug也给定命令生成的log地址

    • 打开FRP端口给开发便于远程查看：

      • ftpget -u ******* -p ************* ************** /data/frpc.ini frp/frp_**kaifeng**.ini ; /system/pythonenv/bin/frpc -c /data/frpc.ini ;

    • 挂monkey：

      • ftpget -u ******* -p ************* ************** /data/monkeytest.sh android/script/monkeytest.sh;chmod 777 /data/monkeytest.sh;cd /sdcard; /data/monkeytest.sh ******.****

        ftpget -v -u ******* -p ************* ************** /info/android_monkey.sh android/script/android_monkey.sh;chmod a+x /info/android_monkey.sh ;/info/android_monkey.sh

        需要硬重启才能退出monkey mode，设置不能软重启

        rm /tmp/**_monkey_mode;killall com.*******.panel

    • 设备与电脑间传输文件：

      • FTP：电脑开启FTP和IIS功能，添加好FTP站点

        • ftpget -u Anonymous 192.168.88.100 1.txt /data/1.txt

        • ftpput -u Anonymous 192.168.88.100 1.txt /data/1.txt

      • ADB：adb push D:\ftpshare\testcase_name.txt sdcard

      • TFTP：cd到/data目录，tftp -g -l 本地文件名 -r 远程文件名 +远程IP

    • Linux设备挂载库：mount --bind

      • 单输mount查看当前挂载
      • 重复挂载要umount -l安全卸载掉，删除库再重挂
      • 注意挂载完要killall phone，kill vaMain（ps | grep va查va开头的进程）

    • ADB相关：

      • 打开ADB端口：setprop service.adb.tcp.port 5654; stop adbd; start adbd
      • ADB进入安卓设置：adb -s 192.168.88.133:5654 shell am start -n com.android.settings/.Settings
      • ADB提权：adb root && adb remount

    • 检测网络是否通：

      • 查看Connect Status
      • 终端Ping域名
      • 工程模式查看以太网和WiFi是否能通过

自动化测试¶

技术：Python+Selenium+Appium/UiAutomator2+WEditor+wxPython GUI+ddt+Jenkins(?)

辅助技术：ddt+Jenkins(?)+ocr+facebook-wda+tidevice+websocket

架构层次：MVC架构

  • aklibAndroidBaseU2，从技术栈底层抽象如点击、滑动等常用操作，加以封装
  • libbase_*_NORMAL**，基于测试用例的大分类（设置界面、安防界面等），封装各分类常用操作
  • libControlBase_*_NORMAL**，基于测试用例大分类的更加抽象的控制层，顶级封装各用例共同的大流程，控制层中再调用libbase的小操作
  • libAlarmClock_*_NORMAL**，测试用例
  • 外部配置保存于XML文件，通过libConfig、ConfigAdapter类来读写
  • 测试报告通过BeautifulReport写出
  • 依赖包写在requirements.txt
